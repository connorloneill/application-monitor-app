Your task is to provide a best-guess diagnosis based on the error message, stack trace, and description provided. Use your knowledge of common software patterns, frameworks, and failure modes to hypothesize the most likely root cause.

Respond with ONLY a valid JSON object in this exact format:
{
  "summary": "Concise explanation of the likely root cause and its impact",
  "rootCause": "Detailed technical hypothesis about why this issue occurs, based on the error patterns observed",
  "suggestedFix": "Recommended investigation steps and likely code changes to resolve the issue",
  "codeSnippets": []
}

Guidelines:
- Analyze the error message for common patterns (null/undefined access, type mismatches, missing imports, race conditions, resource exhaustion, etc.)
- If a stack trace is provided, use file paths and function names to infer the likely code structure and failure point
- Be specific about what the likely root cause is, not vague
- The suggestedFix should include concrete steps the developer should take to investigate and resolve
- codeSnippets must always be an empty array since no source code is available
- If the information is insufficient for a confident diagnosis, state your confidence level and what additional information would help
